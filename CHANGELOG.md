# Changelog

All notable changes to NetworkKB are documented here. Versioning follows
[Semantic Versioning](https://semver.org/): **MAJOR.MINOR.PATCH** —
MAJOR for overhauls that change the shape of the product, MINOR for new
features within the current architecture, PATCH for bug fixes and small
polish with no new features.

Each entry is grouped into up to four sections — **New** (didn't exist
before), **Updated** (existing behavior changed), **Fixed** (something
broken now works as intended), **Removed** (deleted or replaced) — so
it's clear at a glance what kind of change each line is, instead of
everything reading as one undifferentiated list.

See also: [TOOL_VERSIONS.md](./TOOL_VERSIONS.md) for the version history
of each individual interactive tool (Design Helper, DNS Builder,
Flashcards, etc.) — those are tracked separately from this file's
product-wide version, since a patch to one tool doesn't necessarily
warrant a product-wide version bump on its own.

## [3.3.0] — 2026-07-21

### New
- **In-app version history viewer** (Tools → 📋 Version history): a dropdown to pick any release, showing that version's New/Updated/Fixed/Removed bullets right in the browser — no need to leave the app to read `CHANGELOG.md`.
- `data/changelog.json` + `scripts/build_changelog_json.py` — `CHANGELOG.md` stays the single source of truth (the normal, human-editable format), and this script parses it into the JSON the browser actually reads. Same one-way build-step pattern as the search index: edit the markdown, run the script, ship the JSON.

### Updated
- Fixed several markdown line-wraps in `CHANGELOG.md` that broke mid compound-word or mid slash-chain (e.g. a line ending in "multi-" with "cloud" starting the next line) — harmless when GitHub renders the markdown (it collapses soft line breaks), but produced a stray extra space once parsed into plain bullet text for the in-app viewer.

  button.

### Updated

  indexed).
- Stale flashcard-count references on the Home page updated to 80.

## [3.1.0] — 2026-07-21

### New
- Global quick-search — press `/` anywhere (outside a text field) to
  open it. Matches page titles, headings, reference-table entries,
  dropdown options, and form labels.
- Multi-word AND matching in search ("SOA record" finds the SOA panel
  even though that phrase never appears verbatim).
- Search result ranking that weights real headings above incidental
  table/label mentions.
- Permanent search-limitations disclaimer in the search overlay footer.
- `data/search-index.json` + `scripts/build_search_index.py` — the
  index is a static file, regenerated by the script after any content
  edit; not built automatically on page load.
- Version badge in the header, plus this changelog format itself and
  [TOOL_VERSIONS.md](./TOOL_VERSIONS.md) for per-tool version tracking.

### Updated
- Top nav: 12 items → 8. DNS merged into Fundamentals, Switching into
  Routing, NAT into ACL, Wireless into QoS — same "merge sidebars into
  one umbrella" pattern already used for Security. Each merged topic
  keeps its own home card and its own sidebar group.
- Header split into two rows: branding + search on row one, nav gets a
  full-width row to itself — fixes the recurring nav-overflow problem
  properly instead of continuing to shrink nav-link text.
- Sidebar no longer shows on the Home page — reclaims ~210px of width
  for the home-card grid. Still shows normally, correctly aligned to
  the active umbrella section, on every other page.

### Removed
- The `sb-home` sidebar shortcut list — fully redundant with the home
  cards and global search.
- Standalone DNS / Switching / NAT / Wireless top-nav entries (content
  preserved, just relocated under their merged umbrella).

## [3.0.0] — 2026-07-21

### New
- **Fundamentals section**: the OSI model — all 7 layers with a visual
  stack diagram, encapsulation/de-encapsulation walkthrough with a
  worked example, layer-by-layer deep dive, OSI vs TCP/IP comparison,
  troubleshooting mapped to each layer.
- **DNS section**: how resolution works (recursive vs iterative, with a
  diagram), every major record type with real examples, and the
  interactive DNS record builder tool.
- **Security section additions**: NGFW (App-ID/DPI/IPS/TLS inspection)
  and Cisco ISE (802.1X/MAB, posture, TrustSec/SGT) reference content.
  No configuration tools for these two, by design.
- **Design Helper**: perimeter firewall, DMZ zone, and Wireless LAN
  Controller as optional appliances, with a strict bordered
  External/DMZ/Internal visual split and matching device-click modals.
- 18 new flashcards (OSI ×6, DNS ×6, Security ×6) — 74 total.
- `CHANGELOG.md` and an in-app "What's new" card on the Home page.

### Updated
- Design Helper: layer-toggle legend moved to a vertical side column;
  requirements form and detail cards became collapsible.
- Top nav: VPN merged into Security (IPsec is a security topic either
  way) after the section count started genuinely overflowing the nav
  at 1280px — confirmed by measurement, not assumption.
- Home page hero pills and card tags updated to match the new sections
  and flashcard count (several were stale at "56 cards").

### Fixed
- Design Helper: the Internal zone's bordered background started 3px
  below the first device row's actual top edge, so that row visually
  poked through the border. Now has clean 12px clearance.
- Design Helper: a device-icon glyph referenced `currentColor` with no
  color context set, silently rendering the wrong color.

## [2.2.0] — 2026-07-20

### New
- Cisco banner (motd/login/exec) generator, with live
  delimiter-collision detection — warns and auto-swaps the delimiter if it
  appears inside the message text instead of silently generating a
  broken banner.
- Collapsible "How to use this tool" hint on every tool panel.
- Collapsible sidebar, state persisted via `localStorage`.

### Updated
- ACL Builder and Config Validator moved fully into the Tools section
  (previously lived in both Tools and the ACL section's own sidebar).
- Flashcards moved off the top nav into Tools, alongside Quick
  Reference.
- Top nav: Tools moved to the end of the list.

### Fixed
- Flashcards: grading buttons used to look clickable even before a
  card was flipped, but did nothing when clicked in that state — a
  dead click that read as a broken tool. (Superseded by the v1.2.0
  flashcard redesign in the next release — see
  [TOOL_VERSIONS.md](./TOOL_VERSIONS.md).)
- Flashcards: if `data/flashcards.json` fails to load, the card area
  now explains why instead of sitting on "Loading…" forever.

## [2.1.0] — 2026-07-19

### New
- Interactive Design Helper: suggested topology, device/port counts, a
  VLAN plan, and a starter IOS config generated from site requirements.
- Visual SVG topology diagram for the Design Helper — toggleable
  layers, clickable device icons opening a device-config viewer.
- STP deep-dive panels (flavors/PVST+/RSTP/MST, port states and roles,
  protection features, root bridge design) and a full Network Design
  reference section.
- Inline SVG diagrams for ACL in/out direction and NAT inside/outside
  local/global addressing.
- Save/load designs via a PHP + SQLite backend.

### Updated
- **Architecture**: split the single 300KB `index.html` into
  `index.html` + `css/style.css` + focused JS modules + JSON data files
  + the PHP API — enables permanent Docker hosting instead of only
  offline single-file use.
- 113 official reference links (Cisco/IETF/IEEE/NIST) added across
  existing panels.

### Fixed
- OSPF cost calculator: a duplicate `id="ospf-cost"` meant
  `getElementById` returned the wrong element; renamed the output span
  and fixed the divide-by-zero case.
- Flashcards/Quick Reference not displaying: the page-navigation
  function only ever activated the content panel for the `home` page.
  Generalized to any page whose id matches a panel.
